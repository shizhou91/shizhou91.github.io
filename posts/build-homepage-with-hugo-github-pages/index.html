Build homepage on Github pages with Hugo Framework

Instruction#

Here is how to setup a homepage hosted in the Github pages with Hugo.

Install Hugo#

Please follow this link to install Hugo. My platform is Ubuntu and I use the following command line installing Hugo

sudo snap install hugo

Create the static website#

Use the hugo command to create your static website.

hugo new site homepage --format yaml

Run the below command to have a test locally.

hugo server

Create a homepage in Github#

The benifits of deploying the static website are:

  • It’s free. It’s not required to pay for the cloud servive to host your homepage.
  • The website response is fast. We leverage the Github to server our pages. Based on my experience, it’s pretty fast. Sometime even better than the cloud service.

Here are details to how to create a github page.

  • Step 1: Generate Github page with this instruction: https://pages.github.com/. It takes a few minutes to create a static website skeleton.

  • Step 2: Then we could set this Repo to as hugo workspace submodule. For my preference, I like to create a git repo for the hugo workspace. And use add the github pages as a git submodule.

Here is the command line how to add submodule to current git repo.

git submodule add your_git_reps_address public

Generate the website#

Use the hugo command line to generate the webiste. And push to it the git repo.

Check your website for a while after they are all submitted. I mean both the hogo workspave repo and generated public folder.
